# Xmind MD 格式規格

> 本格式定義 Xmind 資料以 Markdown 表示的慣例，供工具輸出與人工編輯後再轉回使用。

---

## 一、整體結構

```markdown
---
source: 原始檔名.xmind
---

# 根節點標題

## L1 節點
### L2 節點
#### L3 節點
- L4 節點
  - L5 節點（縮排兩格）
    - L6+（再縮排）

---

## 圈選框（Boundaries）

| 標題 | 所在位置 | 涵蓋節點 |
|------|---------|---------|
| 圈選框標題 | 父節點路徑 | 節點A / 節點B |
```

---

## 二、層次對應規則

| Xmind 深度 | MD 表示方式 | 範例 |
|-----------|------------|------|
| 0（根節點） | `#` | `# ＢＤ２` |
| 1（L1） | `##` | `## 養成系統` |
| 2（L2） | `###` | `### 角色` |
| 3（L3） | `####` | `#### 數值` |
| 4+（葉節點） | `- 縮排列表` | `- 等級` |

> **原則**：有子節點的深層節點維持標題，無子節點的葉節點改用列表項目。  
> 目前工具統一以深度判斷，後續可依需求調整。

---

## 三、備註（Note）表示

| 情境 | 表示方式 |
|------|---------|
| 深度 0–3 的節點備註 | `>` blockquote，緊接在標題下方 |
| 深度 4+ 的節點備註 | `← 備註內容`，追加在列表項目後 |

範例：
```markdown
#### 技能（服裝）
- 服裝等級  ← 消耗：票卷
- 三顆潛力  ← 消耗：女神之淚

### 養成系統
> 核心目標：讓數值更高
```

---

## 四、圈選框（Boundary）表示

### 4.1 行內標記

圈選框標題附在**所屬父節點**的標題後，以 `〔〕` 標記：

```markdown
### 傷害架構  〔養成要素〕
#### 傷害類型  〔吃「面板」〕
```

### 4.2 彙整表（文件末尾）

所有 boundary 彙整為表格，說明涵蓋範圍：

```markdown
## 圈選框（Boundaries）

| 標題 | 所在位置 | 涵蓋節點 |
|------|---------|---------|
| 養成要素 | ＢＤ２ > PVE | 裝備 / 陣容 |
```

---

## 五、比對差異標記

平行區塊的差異使用備註加 `⚠` 前綴標記：

```markdown
#### 傷害類型
> ⚠ 比較 PVP：無固定傷害
- 物攻 / 魔攻

#### 傷害類型  〔吃「面板」〕
> ⚠ 比較 PVE：多固定傷害
- 物攻 / 魔攻
- 固定傷害  ← ⚠ PVP 限定，PVE 無此類型
```

---

## 六、與 JSON 的對應

| MD 元素 | 對應 JSON 欄位 |
|---------|--------------|
| `#` / `##` / `###` / `####` / `- ` | `title` |
| `> 備註` / `← 備註` | `note` |
| `〔標題〕` | `boundaries[].title` |
| 彙整表的涵蓋節點 | `boundaries[].range` → 子節點名稱 |

| 方向 | 工具 |
