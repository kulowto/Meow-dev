# 線稿提取技法：Color Dodge 邊緣提取

> 此文件記錄 chr_contourLine 工具背後的技法邏輯。
> 接手 AI 在修改參數或調整步驟前，必須先閱讀此文件，
> 確保修改方向符合技法設計意圖，而不是盲目調數值。

---

## 為什麼不用舊版的 Levels 白場拉低？

舊版做法是去色後直接把 Levels 白場拉低（例如 180），靠「角色填色區亮度高、描邊亮度低」的差異來提取線條。

這個方法的根本限制是：**它依賴原圖本身的描邊對比**。如果角色的描邊顏色偏淺、或填色區有暗色漸層，舊版做法要麼留下大量殘色，要麼線條斷掉。

Color Dodge 技法不依賴原圖的描邊顏色，它是**計算出邊緣**，理論上對所有角色圖都有效。

---

## Color Dodge 邊緣提取的核心原理

### 數學邏輯

Color Dodge 的混合公式：

```
結果像素 = 下層像素 ÷ (1 − 上層像素)
```

把去色後的圖放在下層（Normal），把它的反色放在上層（Color Dodge）。

- **在填色平坦區域**：下層是某個灰值 G，上層是它的反色 (1-G)。代入公式：
  `G ÷ (1 − (1−G)) = G ÷ G = 1.0（純白）`
  → 填色區完全變白，在 Multiply 下透明

- **在邊緣和線條區域**：上下層的像素值不是嚴格互補（因為邊緣處有強烈的鄰近像素差異），公式無法整除，產生深色殘留。
  → 線條被保留為深色

這就是為什麼這個技法能「無中生有」地把邊緣線條提取出來，而不需要原圖有明顯的黑色描邊。

## 每個步驟的設計意圖

### Step 1–2：去色 + 複製兩層（chr_desaturate / chr_desaturate_invert）

目的是建立 Color Dodge 所需的「下層 + 反色上層」結構。

去色的原因：Color Dodge 計算需要亮度資訊，去色確保只用灰階亮度，不受色彩通道干擾。

---

### Step 3：反色（Invert）

`chr_desaturate_invert` 是 `chr_desaturate` 的完全反色版本，這是 Color Dodge 邊緣提取的必要條件。

---

### Step 4：Color Dodge 混合模式

讓 PS 即時計算上述公式的視覺結果。此時畫面上大部分是白色（填色區消失），邊緣出現灰色或深色線條。

---

### Step 5：最小值濾鏡（Minimum Filter），半徑 3px

**最小值濾鏡的作用**：把每個像素替換為周圍指定半徑內最暗的像素值。

對線稿來說，效果是：
- 把每條邊緣線**向外擴張 3px**
- 讓本來因為邊緣過渡產生的細灰線變得紮實不斷
- 消除孤立的雜訊像素（孤點周圍都是白，最小值把它推向白）

**半徑 3px 的選擇理由**：
- 半徑過小（1px）：線條可能斷裂
- 半徑過大（4px+）：線條過粗，失去細節，且角落處會有明顯的「方塊感」（Squareness 模式）
- 3px 是使用者確認效果後採用的數值，在一般解析度的遊戲角色圖上表現穩定

**`Roundness` 模式**（圓形擴張）vs `Squareness`（方形）：
使用 Roundness，線條擴張形狀較圓潤，接近手繪描邊效果。

---

### Step 6：Merge Down（向下合併）

**為什麼要合併而不是直接用 Color Dodge 圖層？**

Color Dodge 是一個「視覺效果」，不是真實像素值。如果直接在 Color Dodge 圖層上跑 Levels，調整的是那些「反色像素」而不是我們看到的白底黑線。合併後才是把視覺效果烘焙成真實像素值，後面的 Levels 才能精準地清理線稿。

合併後 `chr_desaturate_invert` 消失，`chr_desaturate` 包含烘焙後的白底黑線像素，重命名為 `chr_contourLine`。

---

### Step 7：Blend If This Layer 白場 245

**操作**：右鍵圖層 → 混合選項 → 混合範圍 → 目前圖層，白場設為 245。

**為什麼 Levels 之後還需要 Blend If？**

Levels 把亮部推向白色（RGB 255），但「白色像素」在 Photoshop 中仍然是**不透明的**。在 Normal 混合模式下，它們會蓋住下方圖層的顏色。

Blend If This Layer 白場 245 的效果：
- 亮度 245–255 的像素 → 完全透明（從圖層合成中移除）
- 亮度 0–244 的像素 → 保持原有不透明度

這讓線稿背景真正「消失」，不是白色覆蓋，是**沒有貢獻**，效果遠比靠 Levels 推白乾淨。

**參數 245 的意義**：
- 設太高（接近 255）：少數極亮的雜訊仍保留，效果不夠乾淨
- 設太低（240 以下）：可能吃掉線條邊緣的灰色過渡，線條顯得生硬
- 245 是保留線條過渡灰的同時清掉背景的平衡點

**編碼邏輯（給 AI 維護用）**：
PS 的 Blend If 滑桿值是 16 位元整數，非分割滑桿時：`value = (white_point << 8) | white_point`。
245 → `(245 << 8) | 245 = 62965`。

---

### Step 8：Levels 清理（黑場 150 / 白場 240 / Gamma 0.7）

合併後的圖層有很多灰色雜訊（邊緣過渡帶來的半透明灰），需要 Levels 清理。

| 參數 | 預設值 | 作用 |
|------|--------|------|
| 黑場 | 150 | 亮度低於 150 的像素全推成純黑，確保線條清晰 |
| 白場 | 240 | 亮度高於 240 的才算純白，稍微保留接近白的灰 |
| Gamma | 0.7 | 中間調整體壓暗，讓線條更濃、灰色雜訊壓掉 |

**這些參數依圖片品質不同需要調整**（見下方）。

## 參數調整指南（給接手 AI）

### 何時需要調整 Levels 參數

| 現象 | 可能原因 | 調整方向 |
|------|---------|---------|
| 線條之外還有大量灰色殘留 | 黑場太低，雜訊沒被清掉 | 把黑場從 150 往上調（試 170–200） |
| 線條斷裂或太細 | 黑場太高，把線條邊緣也推掉 | 黑場往下調（試 120–140） |
| 整體線條太淡 | Gamma 太高 | Gamma 從 0.7 往下調（試 0.5–0.6） |
| 線條太濃，角色細節消失 | Gamma 太低或黑場太高 | Gamma 往上調（試 0.8–1.0），黑場降低 |
| 亮部區域有污點 | 白場太低，正常灰被推成深色 | 白場往上調（試 245–255） |

### 何時需要調整最小值半徑

| 現象 | 調整方向 |
|------|---------|
| 線條在複雜細節處斷裂 | 最小值半徑從 2 調到 3 |
| 線條過粗，角色細節模糊 | 最小值半徑從 2 調到 1 |

## 接手 AI 的操作規範

1. 修改參數前先讀此文件，理解每個參數影響的是哪個技術環節
2. 不要同時調整多個參數，每次只動一個，讓使用者確認效果
3. 使用者回報「效果很好」後，把該參數組合加入「已驗證組合」表格
4. 如需修改步驟順序（例如把 Minimum 移到 Invert 之前），要先確認不會破壞 Color Dodge 的數學前提
