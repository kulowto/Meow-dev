# 影片轉工具開發流程規範 v0.1

適用範圍：以教學影片為來源，轉換為可開發的工具規格或自動化腳本的任務
工具依賴：MEMO.AI（文字轉錄）+ AI 解析
不限軟體：PS / AI / 任何有操作步驟的教學影片皆適用

---

## 概述

本規範定義一套標準化流程，讓 AI 從短影片教學中自動萃取操作步驟、轉換為工具開發規格、並實作為可執行的自動化工具。

核心概念：
- **影片** 是需求來源，不是文件
- **MEMO.AI** 是人工字幕的替代品，解決 AI 無法直接「看影片」的問題
- **AI 解析** 負責修正語音辨識誤差、結構化步驟
- **使用者確認** 是品質把關的唯一節點

---

## 工作流程（5 個 Phase）

### Phase 1｜影片來源確認

輸入：YouTube 連結 或 本機影片檔案

AI 執行：
1. 開啟 YouTube 頁面截圖（Playwright）→ 識別軟體介面（PS / AI / 其他）
2. 讀取頁面 metadata → 取得標題、時長
3. 截圖確認第一幀畫面內容與燒入字幕

輸出：影片資料卡
```
軟體：PS / AI / 其他
標題：
時長：
初步判斷：是否適合製作工具（見篩選規則）
```

---

### Phase 2｜MEMO.AI 文字轉錄

**現行流程（AI 透過 CDP 自動操作）**：
1. 以 CDP 模式啟動 MEMO.AI：
   `Start-Process "C:\Users\{username}\AppData\Local\Programs\Memo\Memo.exe" -ArgumentList "--remote-debugging-port=9222"`
2. CDP 連線：`http://localhost:9222/json` 取得 WebSocket URL
3. 填入 YouTube URL → 點擊第一個「確認轉錄」按鈕
4. **等待對話框出現後，依序設定以下三項（必填，不可略過）**：
   - 模型：**高品質 - Medium**（下拉選單，選錯會影響辨識品質）
   - 運算方式：**GPU 加速**（勾選，有顯卡時速度較快）
   - **自動斷句**：勾選（影響逐字稿分段品質）
5. 點擊對話框內的「確認轉錄」→ 等待轉錄完成
6. 切換至「語音轉文字」tab → 提取逐字稿文字 → 存 `.txt`

**⚠️ CDP 腳本對話框設定注意事項**：
對話框內的模型選單和勾選項必須在 `DIALOG_CONFIRMED` 前完成設定。
相關腳本：`C:/Users/AkatsukiNeko/AppData/Local/Temp/memo_batch.py`

**重要限制**：
- MEMO.AI **僅能轉錄語音**，無法擷取影片中的純文字畫面
- 若影片無口述語音（全靠字幕文字呈現步驟），需改用其他工具或使用者手動補充
- 需改用工具時，在工具規格草稿的來源欄位標注「無語音，需手動補充」

輸出：原始逐字稿（包含語音辨識錯誤，此為正常現象，Phase 3 修正）

### Phase 3｜AI 步驟萃取與清洗

輸入：原始逐字稿

AI 執行四項解析：

**1. 語音辨識修正**
對照「常見辨識修正對照表」（見附錄）修正誤辨字。
無法判斷時標記 `[辨識存疑]` 並保留原文。

**2. 過濾非操作性內容**
- 刪除：開場白、閒話、CTA（記得訂閱、點讚等）
- 保留：含動詞的操作句（「打開」「選取」「添加」「設定」「拉」「拖」「複製」「刪除」）

**3. 步驟結構化**

每個步驟格式：
```
步驟 N：[操作動詞] [目標] [參數或方向（若有）]
原文：[對應原始辨識文字]
```

**4. 效果原理標注**
若影片說明技術原理（如「減藍 = 加黃」），單獨記錄為「效果原理」欄位。

輸出：結構化步驟草稿（等使用者確認）

---

### Phase 4｜使用者確認 + 規格定稿

使用者確認清單：
- [ ] 步驟順序是否正確
- [ ] 有無遺漏的步驟
- [ ] 可調整參數的預設值與範圍
- [ ] 工具用途說明（什麼情況下使用）

輸出：定稿步驟清單 + 可調參數表（見「輸出文件格式規範」）

---

### Phase 5｜工具實作

依定稿步驟，按目標平台規範實作：

| 目標 | 實作方式 | 平台限制 |
|------|---------|---------|
| PS 效果工具 | Python + photoshop-python-api | Windows Only |
| AI（Illustrator）工具 | 尚無 Python COM 方案 | TODO |
| 通用腳本 | 依目標平台決定 | — |

實作流程遵照「程式開發協作規則」：
1. Playground 單元測試 → 通過後整合主流程 → 實機測試視覺確認

## 篩選規則

### 適合製作成工具的

- 有明確步驟序列（動詞性操作，順序固定）
- 參數有具體數值，或可設合理預設值
- 效果可程式化重現，不依賴「靠感覺調整」
- 軟體為 PS（目前唯一有 Python COM 支援的工具）

### 暫時跳過的

| 情況 | 原因 |
|------|------|
| AI（Illustrator）影片 | 無 Python COM 支援 |
| 純手感操作（「拉到你滿意為止」） | 無法程式化 |
| 過於依賴特定素材的步驟 | 無法泛化 |
| 逐字稿還原不出完整步驟 | 資訊不足，需補充 |

## 輸出文件格式規範

每支影片完成 Phase 3 後，產出「工具規格草稿」：

```markdown
# 工具規格草稿：[效果名稱]

## 來源
- 影片：[URL 或檔名]
- 軟體：[PS / AI / 其他]
- 時長：[秒數]

## 效果描述
[一句話描述效果與使用情境]

## 操作步驟
1. 步驟一
2. 步驟二
...

## 可調參數
| 參數名稱 | 預設值 | 範圍 | 說明 |
|---------|--------|------|------|

## 效果原理
[技術說明，如「減藍 = 加黃」]

## 開發狀態
- [ ] Phase 3 完成（步驟草稿確認）
- [ ] Phase 4 完成（使用者確認定稿）
- [ ] Phase 5 完成（實作 + 實機測試）
```
