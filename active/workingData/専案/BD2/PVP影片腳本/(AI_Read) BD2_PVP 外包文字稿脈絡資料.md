## 這份文件的用途

這份文件保存的是「BD2 鏡中之戰 PVP 教學」這個外包文字稿專案的**專案特定脈絡**。

這些內容**不適合直接放進文案領域文件**，理由有兩種：
1. **遊戲特定內容**：跟 BD2 這款遊戲的機制、等級系統、角色設計直接相關，換了任何一個專案就不適用
2. **外包工作流程**：是這個合作中的操作規範（節點標記、備註格式等），不是通用的文案撰寫規則

**AI 讀到這份文件時的使用方式**：
- 執行 BD2 系列的文字稿任務時，作為補充規格參考
- 搭配領域文件（`(AI_Read) 文案領域文件.md`）一起使用：**領域文件提供通用規則，本文件提供專案特定細節**
- 不要把這份文件的規則當作所有文案任務的標準

## 一、平台版本策略

| 平台 | 長度目標 | 做法 | 目的 |
|------|---------|------|------|
| BiliBili | 2～4 分鐘 | 單集完整版，直接上傳 | 主力收益平台，門檻較低 |
| YouTube 長片 | 10 分鐘以上 | 同類型數集串接（去掉各集的過渡收尾段） | 滿足 YT 長片演算法 |
| YouTube Shorts | 60 秒以內 | 以「陣容介紹」為主，做宣傳導流用途 | 吸引新觀眾點進長片 |

> 同一批素材需要剪成兩個版本（BiliBili 版 + YouTube 長片版），YT Shorts 另外獨立製作。

## 二、受眾分層（依遊戲進度）

BD2 受眾不是同質群體，同一個知識點對不同進度的玩家有不同意義：

| 族群 | 階段目標 | 對應內容方向 |
|------|---------|------------|
| 新手沒資源 | 基礎觀念建立 | PVE 入門、傷害計算、角色定位 |
| 能穩定 PVE 魔獸 10，想踏入 PVP | 入門陣容選擇與資源規劃 | 低造價陣容、資源優先順序 |
| PVP 穩定紅寶石 → 藍寶石 | 對戰思路與陣容細化 | 陣容優劣勢、站位基礎 |
| PVP 穩定藍寶石 → 大師 | 進階站位、技能順序優化 | 卡平 A 機制、對策思路 |
| 宗師以上 | 非主要受眾 | 通常自行研究，不需要針對這層特別設計 |

**受眾切入方式（依玩家類型）**：

| 玩家類型 | 切入方式 |
|---------|---------|
| 角色黨（喜歡特定角色） | 從角色定位切入，推薦可搭配的陣容；適合做成封面圖或單集說明 |
| 強度黨（追求競技強度） | 從陣容強度切入，逐項說明構築細節與技能順序 |

## 三、BD2 陣容類影片結構（類型 B）

> 此結構是「陣容介紹型影片」的專用格式，針對 BD2 的遊戲機制設計。

**骨架**：核心概念 → 陣容資訊 → 陣容細節

### ① 核心概念
一句話說明這個陣容的打法邏輯：
- 強攻隊：首回合直接減員，創造人數差取得優勢
- 雙嘲隊：首回合疊 Buff 撐過去，下回合爆發

### ② 陣容資訊

| 項目 | 說明 |
|------|------|
| 角色 | 哪五隻，哪些位置有替代空間 |
| 硬需求 | 哪些角色有特定潛力、服裝突破等硬需求（常見誤區） |
| 技能施放順序 | 角色 A → 角色 B → 角色 C（建議做一張順序圖） |
| 裝備搭配 | 血量、物抗、魔抗、皮膚綁定建議 |
| 常見站位 | 站位說明 + 原因 |
| 新手參考 | 強度評估、造價、階段性目標 |

### ③ 陣容細節
- 哪些角色是核心不能拆
- 哪些角色目前無替代，原因是什麼
- 對上哪些陣容有優勢？哪些有劣勢？

## 四、外包工作流程規範

這些是這個合作中使用的操作格式，用來對齊創作方與剪片方。

### 節點標記規範

在腳本中標記段落跳轉點，讓觀眾能快速跳到想看的段落。

格式：
```
- 段落主題名稱
```

> 時間戳由蘑菇組負責填入，創作方只需標記段落名稱即可。

### 備註標記規範

影片中需要插入額外素材（圖片、連結等）時，在腳本對應位置標記：

```
<##備註：[圖片 / 連結]，類型說明與原因##>
```

範例：
```
<##備註：陣容站位圖，說明前排後排的位置分配##>
```

## 五、額外製作物

### 一圖摘要（每集創作方負責）

每集影片額外製作一張摘要圖，用於社群分享：
- 重點關鍵字、議題摘要
- 輔助插圖
- 可點選的影片連結或 QR Code

## 六、導流策略

影片之間可以相互導流，但**不能過於頻繁**（容易讓觀眾反感）。

做法：在提到相關主題時，口頭說明並附上連結。  
範例：「這個陣容對上水人隊是劣勢，不熟悉的朋友可以參考影片附上的連結。」

## 七、背景資料：收入模式分析

> 這是決策背景知識，說明為什麼選擇這樣的平台策略。

| 模式 | 說明 | 前提條件 |
|------|------|---------|
| 平台分潤 | 依流量獲得平台收益 | 門檻高、額度有限 |
| 業配分潤 | 廠商買影片時間做廣告 | 需先有流量 |
| 代操剪片 | 幫他人剪輯換收入 | 需有作品集曝光 |
| 本業擴展 | 用短影音行銷，吸引本業客戶 | 最主流、門檻最低 |

對於這個系列：以 BiliBili 的短視頻收益為主，YouTube 作為長期佈局，YT Shorts 做流量入口。
