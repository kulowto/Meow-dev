## 一、每集影片的通用結構

```
開頭（前 5～10 秒）→ 主題內容1（開頭總結 → 說明）→ 主題內容2 ... → 收尾
```

---

## 二、開頭：前 5～10 秒

**目的**：讓觀眾先知道這集會看到什麼，確保他們有興趣繼續看下去。

從以下四種方式擇一作為開頭：

| 類型 | 範例 |
|------|------|
| **破題** | 「PVP 新手最常踩的誤區是什麼？」 |
| **重點** | 「這集幫你解析 PVP 常見隊伍，找到適合自己的陣容」 |
| **觀點** | 「你相信的 XXX 打法，其實不是個好方法嗎？」 |
| **反問** | 「你還在用這個方法嗎？現在有更好的辦法了」 |

---

## 三、開場總結（接在開頭後）

用一句話說明：**這集的目的是＿＿＿**

---

## 四、主題內容：依影片類型製作

### 類型 A｜教學類

**結構**：觀念 1 → 觀念 2 → 觀念 3 → ...

每個觀念的展開方式：
- **標題**：一句話總結這個觀念
- **誤區**：如果有常見錯誤，先說（更抓注意力）
- **說明**：解釋原因
- **舉例**：用生活案例類比，讓不熟的人也能理解

**依受眾分層給目標**：

| 族群 | 階段目標 |
|------|----------|
| 新手沒資源 | 基礎觀念建立 |
| 能穩定 PVE 魔獸 10，想踏入 PVP | 入門陣容選擇與資源規劃 |
| PVP 穩定紅寶石 → 藍寶石 | 對戰思路與陣容細化 |
| PVP 穩定藍寶石 → 大師 | 進階站位、技能順序優化 |
| 宗師以上 | 非主要受眾，這層玩家通常已自行研究 |

---

### 類型 B｜陣容類

**結構**：核心概念 → 陣容資訊 → 陣容細節

**① 陣容核心概念**
- 一句話說明這個陣容的打法邏輯
- 範例：
  - **強攻隊**：首回合直接減員，創造人數差取得優勢
  - **雙嘲隊**：首回合疊 Buff 撐過去，下回合爆發

**② 陣容資訊**

| 項目 | 說明 |
|------|------|
| 角色 | 哪五隻，哪些位置有替代空間 |
| 硬需求 | 哪些角色有特定潛力、服裝突破等硬需求（常見誤區） |
| 技能施放順序 | 角色 A → 角色 B → 角色 C（建議做一張順序圖） |
| 裝備搭配 | 血量、物抗、魔抗、綁定皮膚建議 |
| 常見站位 | 原因說明 |
| 新手參考 | 強度評估、造價、階段性目標 |

**③ 陣容細節**
- 哪些角色是綁定的？不能拆
- 哪些角色目前無替代、原因是什麼
- 對上哪些陣容有優勢？哪些有劣勢？

---

### 類型 C｜採訪類（考慮中）

找深入的玩家一起討論：
- 環境現況與陣容差異
- 個人對策與使用心得

---

## 五、節點標記規範

**目的**：讓觀眾能快速跳轉到想看的段落

**格式**：
```
- 段落主題名稱
```

> 時間戳由蘑菇組的人協助填入，創作方只需標記段落名稱即可。

---

## 六、備註標記規範

影片中需要額外插入圖片或連結時，用以下格式標記：

```
<##備註：[圖片 / 連結]，類型說明與原因##>
```

---

## 七、導流策略

影片之間可以相互導流，但**不能過於頻繁**（容易讓觀眾反感）。

**做法**：
- 在提到相關主題時，口頭說明並附上連結
- 範例：「這個陣容對上水人隊是劣勢，不熟悉的朋友可以參考影片附上的連結」

---

## 八、額外製作物

### 一圖摘要（創作方負責）
每集額外製作一張摘要圖，包含：
- 重點關鍵字、議題摘要
- 輔助插圖
- 可點選的影片連結或 QR Code

### 受眾切入方式

| 受眾類型 | 切入方式 |
|----------|----------|
| **角色黨**（喜歡特定角色） | 從角色定位切入，推薦可玩陣容；可做成封面或單集說明 |
| **強度黨**（追求競技強度） | 從陣容強度切入，逐項說明構築細節 |

---

## 九、平台剪輯版本

| 平台 | 長度目標 | 做法 |
|------|----------|------|
| BiliBili | 2～4 分鐘 | 單集直接上傳 |
| YouTube 長片 | 10 分鐘以上 | 同類型數集串接（去掉各集收尾過渡段） |
| YouTube Shorts | 60 秒內 | 以「陣容介紹」為主，做宣傳導流用途 |
